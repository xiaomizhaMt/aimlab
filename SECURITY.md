# Security Policy

## Supported Versions

This is a personal learning project. Security fixes are handled on the `main`
branch only.

| Version | Supported |
| ------- | --------- |
| `main`  | Yes       |

## Reporting a Vulnerability

If you find a security issue, please open a GitHub issue with a concise
description and reproduction steps.

Do not include passwords, API keys, access tokens, private screenshots, or other
sensitive personal data in reports.

## Scope

This project is intended for local computer-vision experiments and Aim Lab
offline training scenarios. Reports should focus on repository security, unsafe
defaults, dependency risks, or accidental exposure of sensitive files.
